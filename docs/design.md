# Zkユーザーストーリー

## 🧭全体思想
- CLIのようにキー操作だけでノート操作ができる
- 少ないキーに多義的な意味を持たせ、操作を単純にする
- 操作はUIではなく文脈（モード・選択・カーソル）で決まる

## 🧩共通ルール（全モード）
- ノートは必ず既存ノートから派生する
  - 開いているノートが派生元
  - なければモードごとのルートノート
- リンクはObsidian標準機能を利用
- 思考の道筋を残すことを最優先する

## 🧠Coreモード（自分の知識・永続）
- Coreノートは自分の知識を永続的に保存するためのノート
- Coreノートには次の要素をフロントマターに自動付与する
- Cから始まる一意ID、エイリアス
### Coreモードのキー操作
- 何も選択していない
- →新規Coreノートを作成
- ワードを選択している
- →そのワードのCoreノートを開く
- →なければそのワードをタイトルに新規作成
- [[]]内にカーソルがある
- →中のワードを選択扱い（エイリアス除外）で上と同じ挙動

## 📚Refモード（外部知識・参考文献）
### Srcノート（参考文献そのもの）
- 書籍・論文・記事などを表すノート
- BookSearchプラグインで作成
- Refノートの起点となる
- Sから始まる一意のID、エイリアスを持つ
### Refノート（参照箇所）
- Srcノート、またはRefノートからワンキーで作成
- どこから作成しても論理的な派生元は必ずSrcノート
- 参照ページ・章・引用・要約などを書く
- Rから始める一意のID、エイリアスを持つ

## 🗒️Tempモード(一時メモ)
- Tempノートは一時的な思いつき用
- ID・エイリアスは持たない
- Core/Refへの昇格はしない
- 昇格したい場合は新規Coreノートを作る

### 腐敗検知
- 一定時間アクセス・更新されていないTempノートを「腐敗」とみなす
- 腐ったメモはTempルートノートに一覧表示
- 自動削除・自動昇格はしない（判断はユーザー）

## 🔗バックリンク（例外仕様）
- Obsidian標準のバックリンクUIは使わない
- ノート本文にバックリンクを自動追加・更新する
  - 編集したファイルが保存されたタイミングで更新する
### バックリンク仕様
- 1行1リンク
- 表示内容: タイトル+エイリアス
- 並び順: 更新順(新しく触れたノートほど重要)
- 参照リンク(`![[ ]]`)では表示されない
  - 本文だけを表示したいため

## 🆔 ID・エイリアス
- 対象: Core/Ref
- ID:
  - 完全一意
  - フロントマターに自動記載
  - IDの先頭文字でノート種別を表す
    - C: Core
    - S: Src
    - R: Ref
  - 先頭はC/R/Sのいずれか、それ以降は[a-z0-9]の判断つきやすい小文字を使う
- エイリアス
  - ID先頭4文字から開始
  - 重複時は1文字ずつ延長
  - エイリアスも一意
- ノート名・エイリアスは 検索・補完候補
  - obsidian-various-complements-plugin に依存

## ⚙️コンフィグ
- バックリンク、腐敗検知は有効無効を設定できる
  - デフォルトでは有効
- 次のホットキーを登録できること
  - 新規作成・リンク先移動

## 🚫 やらないこと
- モード以外の特別な意味をツール側で定義しない
  - （TODOノート・日記ノート等はユーザー運用）
- Vimのような複雑なキーストロークを用意しない
- UI操作を前提にしない
- 自動分類・自動昇格をしない
- UIでしか辿れない情報を作らない

## 🧱モード構造まとめ

| モード  | 役割    | ID | 昇格 | 起点      |
| ---- | ----- | -- | -- | ------- |
| Core | 自分の知識 | あり | —  | Coreルート |
| Ref  | 外部知識  | あり | —  | Srcノート  |
| Temp | 思いつき  | なし | なし | Tempルート |

## 📝 一文まとめ（自分用）
> 考えるためにTempがあり、
確かめるためにRefがあり、
残すためにCoreがある。
