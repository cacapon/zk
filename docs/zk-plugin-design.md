# ZettelKasten Obsidian Plugin 設計ドキュメント

## コア体験

**「90%キー操作でメモの作成・編集・移動ができる」**

モードを切り替えることで役割の異なるメモ（一時メモ・文献メモ・永久保存メモなど）の作成をスムーズにし、少ないキーバインドで完結する操作体験を目指す。

---

## 概念設計

### モードとは

モード = **箱（ディレクトリ）＋ 運用ルール のセット**

メモの種類（一時保存・参考文献・永久保存など）によって挙動が変わる部分をモードとしてカプセル化する。

### モードの構成要素（Modeオブジェクト）

| プロパティ | 型 | 説明 |
|---|---|---|
| 名前 | Str | モードの識別名（例: Core, Temp, Ref） |
| DirパスPath | Path | そのモードのノートを格納するディレクトリ |
| TempパスPath | Path | 新規Zettel作成時に使うテンプレートファイルのパス |
| CurrパスPath | Path | 現在作業中のZettelのパス（カレント） |

### rootノートとは

モード作成時に必ず用意する、そのモードの**原点となるZettel**。そのモードに関する情報を書き、そこからメモを派生させる運用を想定。

## クラス設計

### ModeListクラス（モード一覧管理）

```typescript
class ModeList {
  modes: Mode[]

  addMode(mode: Mode): bool
  deleteMode(mode: Mode): bool
  getModes(): Mode[]
  updateMode(mode: Mode): void
}
```

### CurrentModeクラス（現在モード管理）

```typescript
class CurrentMode {
  mode: Mode

  setMode(mode: Mode): void
  getMode(): Mode
}
```

## 主要フロー

### 1. モードを作る

1. ユーザー入力: モード名
2. ユーザー入力: Dirパス
3. ユーザー入力: テンプレートパス
4. CurrパスをDirパスに設定
5. ModeListに登録 → `addMode(Mode): void`
6. カレントパスのメモを開く

### 2. モードを消す

1. モード一覧＆スイッチャーを出す
2. 選んだモードを消す → `deleteMode(Mode): void`

### 3. 今のモードを変える

1. モード一覧＆スイッチャーを出す
2. 変更後モードのカレントパスを開く

### 4. Zettelを開く・作る

```
モード選択されている？
  ├─ されていない → モードを変える（スイッチャーへ）
  └─ されている
       ├─ カーソル付近の文字列を選択
       │    ├─ [[abc]]形式 → abcを選択
       │    ├─ 選択テキストあり → abcを選択
       │    └─ 未選択 → NULL
       ├─ StrのZettelが存在する？
       │    ├─ する → StrのZettelを開く
       │    └─ しない → StrのZettelを作る（NULLならデフォルト）
       └─ モードのカレントを変える
```

## キーバインド設計

**必須キーバインドは2つに絞る**

| キーバインド | アクション |
|---|---|
| (割り当て予定) | モードを変える |
| (割り当て予定) | Zettelを開く・作る |

モード作成・削除等はモード変更スイッチャーに内包し、キーバインドを増やさない。

---

## ファイルオープン時の自動追従

Obsidianのファイルオープンイベントをフックして、モード・カレントを自動追従させる。

### 追従ルール

| 状況 | 挙動 |
|---|---|
| モード内の別ノートを開いた | カレントパスを変更する |
| 異なるフォルダのノートを開いた | 対応モードがあればモード変更、なければ未選択にする |

### ズレ検知と確認ダイアログ

Zkアクション実行時に**現在のモードと開いているファイルのフォルダが異なる場合**のみ確認を挟む。

```
「今モードAですが、開いているファイルはモードBのフォルダです。
  どちらで作成しますか？」
  ├─ 今のモードで作成（モードAのまま）
  └─ このファイルのモードで作成（モードBに切り替えて作成）
```

通常操作（モードとファイルが一致している時）はノーストップで動作する。

---

## モード変更の通知

| 方法 | 説明 |
|---|---|
| リボン表示 | 現在のモード名と色を常時表示（例: Core=緑、Temp=黄） |
| Notice | モードが切り替わった際にポップアップ通知 |

---

## 初回起動・モードなし状態

スイッチャーに「モードを新規作成する」の項目のみ表示し、そこから最初のモードを作成できる。

---

## 予定している追加機能

- **自動バックリンク機能**: Obsidianのバックリンクはキー操作に優しくないため、キーで完結するバックリンク表示を実装
- **rootに戻る機能**: 現在のモードのrootノートに素早く戻れるアクション
- **リンクスイッチャー**: 関連リンクへのキー操作での移動

## アーキテクチャ方針

### Clean Architectureを意識した境界設定

試作品の反省を踏まえ、ObsidianのAPIへの依存を局所化する。

```
コアロジック層（Obsidian APIに依存しない）
  ModeList, CurrentMode, Zettel操作ロジック

インターフェース層（境界）
  ファイル操作I/F, 通知I/F, エディタI/F

Obsidian依存層
  実際のAPI呼び出し（Notice, vault, editor など）
```

新機能追加時はコアロジックを触らず、インターフェース越しに拡張できる設計を目指す。

### CI / 開発自動化

- **pre-commitフック（husky）**: コミット前にユニットテスト＋型チェックを自動実行
- **ユニットテスト（Vitest）**: コアロジック層を中心にテストを書く
- **GitHub Actions**: Push時にテストを自動実行

### Claude Codeとの協業方針

- 設計は自分で決める（このドキュメントをベースにする）
- Claude Codeには1機能ずつ小さい単位で依頼する
- 書いてもらったコードは必ず読み、わからない部分は説明させる
- まず `ModeList` と `CurrentMode` のコアクラスから始める
